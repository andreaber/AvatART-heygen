{# Template de perfil de usuario para Gem-AvatART. Este template implementa la página completa de gestión de perfil personal donde los usuarios pueden editar su información personal, cambiar contraseña, subir avatar y visualizar estadísticas. Incluye validación de formularios y previsualización de avatar en tiempo real. El template incluye: - Formulario completo de edición de perfil con WTForms - Sistema de subida y previsualización de avatar - Sección de cambio de contraseña con validación - Información del sistema (fechas, invitaciones) - Estadísticas personales según rol del usuario - Badges de estado y verificación - Validación de errores con mensajes específicos Clases Bootstrap utilizadas: - Layout: container, row, col-*, mx-auto, d-flex, justify-content-between - Cards: card, card-body, card-header - Forms: form-label, form-control, btn - Images: img-fluid, rounded-circle - Badges: badge, bg-primary, bg-success, bg-warning, bg-info - Typography: text-muted, text-center, text-danger, text-primary, text-success, text-info - Spacing: mb-*, mt-*, fa-8x Variables Jinja utilizadas: - current_user: Usuario autenticado con todos sus campos - form: Objeto WTForms con campos del formulario - form.field_name: Campos específicos del formulario - form.field_name.errors: Errores de validación por campo Campos del formulario WTForms: - avatar: Campo de archivo para subir imagen de avatar - first_name: Nombre del usuario - last_name: Apellido del usuario - email: Dirección de correo electrónico - phone: Número de teléfono - current_password: Contraseña actual para verificación - new_password: Nueva contraseña - confirm_password: Confirmación de nueva contraseña - hidden_tag(): Token CSRF para seguridad Rutas Flask referenciadas (url_for): - main.index: Volver a página principal Iconografía Font Awesome: - fas fa-user-edit: Edición de perfil - fas fa-user-circle: Avatar placeholder - fas fa-camera: Cambiar avatar - fas fa-check-circle: Usuario verificado - fas fa-lock: Cambiar contraseña - fas fa-info-circle: Información del sistema - fas fa-chart-bar: Estadísticas del usuario - fas fa-arrow-left: Volver - fas fa-save: Guardar cambios Propiedades del usuario mostradas: - full_name: Nombre completo del usuario - username: Nombre de usuario único - role: Rol del usuario (con capitalización) - status: Estado de la cuenta - is_verified: Estado de verificación - avatar_url: URL del avatar actual - created_at: Fecha de registro - last_login: Último acceso - invited_by: Usuario que lo invitó (si aplica) Estadísticas condicionales mostradas: - reels.count(): Número de reels creados (si puede crear reels) - created_avatars.count(): Número de avatars creados (si puede crear avatars) - commissions_earned.count(): Número de comisiones ganadas Métodos Jinja utilizados: - current_user.role.value.title(): Capitalización del rol - current_user.status.value.title(): Capitalización del estado - current_user.created_at.strftime(): Formateo de fecha de registro - current_user.last_login.strftime(): Formateo de última conexión - current_user.can_create_reels(): Verificación de permisos - current_user.can_create_avatars(): Verificación de permisos Estilos inline utilizados: - display: none: Campo de archivo oculto - max-width: 150px: Tamaño máximo del avatar Funcionalidades JavaScript: - addEventListener('change'): Detección de cambio en input de archivo - FileReader API: Lectura de archivo para previsualización - DOM manipulation: Reemplazo de icono por imagen previsualizada - readAsDataURL(): Conversión de archivo a URL de datos Validación de errores: - Iteración sobre form.field.errors para mostrar errores específicos - Clases text-danger para styling de errores - Validación tanto del lado cliente como servidor #} {% extends "base.html" %} {% block title %}Mi Perfil -
